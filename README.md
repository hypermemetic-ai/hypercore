# hypercore

hypercore is a small design seed for keeping a growing codebase coherent when the
machine starts each session without memory.

The current corpus is intentionally bare. It keeps the durable ideas and drops the old
machinery, historical work, and runtime logs.

Read:

- `hypercore.md` for the compact methodology.
- `intent/organizing-document.md` for the current intent segments.
- `notes/` for preserved decomposition notes that are not current doctrine.
