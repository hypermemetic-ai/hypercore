The fold commit speaks faithfully — a fenced crossing composes a faithful commit subject for a trivial delta, instead of falling back to a crude truncation that leaks the heading marker and collapses the summary.

The first live fenced crossing (`worker.run` on `the-candidate-prompt-speaks-as-the-architect`, fold commit `bfcf9e7`) auto-composed `fold: # the-candidate-prompt-speaks-as-the-architect — t → channels`, against the clean shape a hand-authored fold takes (`fold: the-view-reads-through-the-tree — the view reads open work through the one tree reader → self-model`). Three faults in one subject: the `# ` heading marker leaks into the title; the summary collapses to a single character (`t`); and the capability reads `channels`, but the crossing touched `design`. The cause is the trivial-delta fallback path. `delta.py:245` composes `fold: {subject} → {label}` where `subject = delta.subject or tree._subject(node.text)` and `label = ", ".join(touched) or "channels"`. A *trivial* delta — a below-floor, code-only change like #3 — carries no `delta.subject` and touches no spec capability, so **both** fallbacks fire: `subject` falls back to `tree._subject(node.text)`, which is `" ".join(text.split())[:50]` (tree.py:244) — it leaks the body's leading `# ` and hard-truncates at 50 chars mid-word — and `label` falls back to the literal `"channels"`. The dispatch commit (tree.py:182, `dispatch: {_subject(node.text)}`) shares the `_subject` fault. The fold commit is operator-facing provenance — the durable record of what each crossing did — and AGENTS.md holds that the architect authors every operator-facing word; an auto-composer that mangles the subject corrupts that record on every fenced crossing, the very next one (#2) included.

Build it so the record reads true. The composed `fold:`/`dispatch:` subject of a fenced crossing reads as faithfully as the hand-authored one — the node's title without its heading marker, a real one-line summary of the change, and the capability the fold actually routed to. The fix likely belongs on the *delta*, not the commit composer: a trivial delta should still carry an architect-authored `subject` (and a meaningful capability) so the fallback never fires — the architect authors the fold's operator-facing line the same way it authors a non-trivial delta's body. And if `tree._subject` survives as any commit's subject — it backs nine machine commits (file-intent, spawn, approve, settle, cut, dispatch, recover, raise, hold) — it should at minimum strip the leading heading marker and cut on a word boundary, never mid-token.
