# design-it-twice: The worker fence must isolate the host at the OS l [machine]

A load-bearing interface for 'The worker fence must isolate the host at the OS l' was designed twice — candidates bubblewrap, landlock, container, ports-adapter, each fenced, each built to a different brief — and compared on depth, locality, and seam placement.

design-decision: The worker fence must isolate the host at the OS l → hybrid — Every candidate agrees on the chokepoint, and they are right to: the seam is `worker_transport`, the one funnel every live worker passes through, with the mechanism strictly below it. The real contest is the *shape* of that below-seam cut. Two shapes actually compete — a pure argv-transform (bubblewrap) and a mechanism-agnostic port carrying a child hook (ports-adapter). landlock and container are the same function-pair seam bound to a specific, weaker mechanism, and the container author self-eliminates.

**Depth.** bubblewrap's `jail(argv, worktree, store) -> list[str]` is the deepest single move because it is *pure*: it builds the jail argv and spawns nothing. Purity buys two properties no other candidate gets for free. (1) The live fence and the gate's write-probe run the *identical* code path — which is the only structural reason the gate validates what production actually runs. The precise defect behind the 2026-06-24 escape was a fence that was *named* but was not the thing under test; purity makes "named" and "tested" the same object. (2) The scaffold check can assert the jail's *shape* with neither bwrap nor git installed. ports-adapter hides marginally MORE — it normalizes argv-prefix and child-side-syscall mechanisms into one `Command` — but that extra hiding is machinery for Landlock, the mechanism the contest did not pick.

**Locality.** Both are transport-local. ports-adapter's re-contest cost (rewrite one adapter, zero transport diff) is only marginally better than bwrap's (swap one function body; "bwrap" appears in exactly one function; the transport line is unchanged). That marginal gain does not pay for a new Protocol + `Command` dataclass + file when exactly one mechanism is ever live on a host and it is swapped by rewrite, not by runtime dispatch. Tellingly, two of the three mechanism authors (landlock, container) independently converged on the function-pair seam and explicitly judged the Protocol unnecessary.

**Seam placement.** ports-adapter is the most principled in the abstract: the seam sits exactly at "what the jail guarantees" vs "how this host imposes it," and `command -> command` carrying `preexec` is the one cut that admits a child-side mechanism (Landlock) without a re-exec trampoline. That is the literal lesson of the escape — don't let the seam encode an assumption about the mechanism (the old `cwd=` encoded "a directory is a jail"). The argv-prefix shape does re-encode "the mechanism is a command prefix": true for bwrap, true for Landlock only via a relauncher. So ports-adapter has the better-placed seam in principle — but it pays for that by building the polymorphic machinery before the second real instance has landed.

**Why the hybrid is deepest.** Take bubblewrap's pure-transform spine — the code-path identity and the free shape-check are non-negotiable given the gate's history — and graft on the two things ports-adapter does better that are NOT speculative. First, the probe returns a *reason*, `jail_available() -> str | None` (None = can enforce, else a one-line reason), not a bare bool: the brief explicitly demands "a clear skip or failure … rather than passing silently," and a reason string is what makes the skip loud. Second, the negative-space invariant — there is no no-op jail; absence can only be a non-None reason, never a silent pass — which is the structural fix for the exact failure that let the escape land. I deliberately do NOT pull in the `Command`/`preexec` port now: the chosen mechanism is argv-prefix, `jail`'s `argv -> argv` shape already admits a relauncher-Landlock if a re-contest picks one, and a port for a variation realized by a single live mechanism is abstraction ahead of its second instance. Build the port the day a second live mechanism arrives; until then this is the most behavior behind the smallest surface, kept where it belongs.

## Grounds

- **bubblewrap — pure `jail(argv, worktree, store) -> list[str]` + `jail_available() -> bool`, transport signature unchanged, `_store` private.**: Deepest single move via purity: the live fence and the gate's write-probe share the exact code path (the only reason the gate tests what production runs), and the scaffold check asserts the jail's shape with no bwrap/git installed. Mechanism named in exactly one function; transport-local. Weaknesses: the probe is a bare bool, dropping the reason the gate needs to skip loudly; the `argv -> argv` shape re-encodes "mechanism = command prefix" (survivable — Landlock fits via a relauncher — but it is the assumption-leak class that produced the escape). Chosen as the hybrid's spine.
- **landlock — boundary byte-identical `worker_transport(cwd) -> (prompt)->str`, `landlock_available() -> tuple[bool, str]`, self-restricting `--jail` relauncher below.**: Strongest "the interface did not grow by one character" argument and a correct transport-local seam; its `tuple[bool, str]` probe carries the reason, adopted in spirit by the hybrid. But it binds the interface to the contest's non-winner and imports a stricter host precondition (kernel ≥ 5.13, Landlock enabled) for no guarantee bwrap doesn't already deliver against a filesystem-only fence. Right seam, wrong mechanism.
- **container — `worker_transport(worktree, argv, *, env) -> WorkerProc` + `fence_available() -> FenceReady | FenceAbsent`, rootless container below.**: Self-eliminates honestly: the seam is deep and mechanism-independent, but the container is dominated — identical guarantee against a filesystem-only fence, strictly more weight (a runtime plus a pinned image artifact to build/version/pull, hundreds of ms–s startup on every spawn's hot path), and its one edge (pid/ipc/user-ns isolation) is out of scope by the brief. Widest surface of the four (a `WorkerProc` class, changed signature). Confirms the function-pair seam; does not earn its mechanism.
- **ports-adapter — `Fence` Protocol (`enter(worktree, command) -> Command`, `available() -> str | None`) + `Command` dataclass carrying `preexec`, in a new `transport/fence.py`.**: Most principled seam: placed exactly at guarantee-vs-mechanism, with `command -> command` carrying `preexec` admitting child-side Landlock without a trampoline, and a negative-space property (no null fence; absence = non-None reason) that structurally bars the silent-pass that let the escape land. Its probe-reason and negative-space discipline are pulled into the hybrid. Not chosen whole: the Protocol + `Command` + new file is polymorphic machinery for a variation realized by one live mechanism swapped by rewrite, not runtime dispatch — abstraction ahead of its second instance. Revisit if a re-contest lands a child-side mechanism.
