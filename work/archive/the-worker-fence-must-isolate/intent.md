The worker fence must isolate the host at the OS level — today it does not, and a live worker has already written the main tree. intent.md §74 promises an OS-enforced fence: a worker 'runs in its own git worktree, fenced by the operating system from its siblings and from the main line ... the worker's own worktree is writable, the rest of the host read-only, and the shared git history writable so the worker's own commits reach the one record' — 'the fence is the system's, not a convention a worker could break by accident.' The engine never built this: `worker.worktree` only `git worktree add`s a separate checkout, and `transport.worker_transport` sets the worker subprocess's cwd to that checkout — but cwd is a starting directory, not a jail, and the omp/gpt-5.5 worker is a coding agent that reaches the whole filesystem. worker.py admits it in a parenthetical: 'intent.md's full fence — the rest of the host read-only — is the system's to enforce around this; the worktree is the seam.' On 2026-06-24 the first live worker, dispatched on vocabulary-check with its cwd at its worktree, edited 11 tracked files on main plus a new engine module — the watched evidence the README's honest-limit discipline said the first autonomous run would produce: the fence is not there.

Make the fence real: a live worker cannot write any path outside its own worktree — an attempt to modify the main tree, a sibling's worktree, or the host fails at the OS level, not by convention — while its own worktree stays writable and the shared git object store stays reachable so its commits still reach the record. §74 keeps the net open (a worker thinks over the net; spend/publish/pull is caught at the decisions floor, not walled off here); the filesystem and the shared .git are what must be fenced. The enforcement wraps the worker's transport (`transport.worker_transport`), which spawns `omp -p` at the worktree today and must instead spawn it inside a real jail. The host is Linux. spec/worker.md's 'a worker runs fenced in its own git worktree' requirement and intent §74 carry the change; the README's honest-limit note is corrected by whatever lands.
