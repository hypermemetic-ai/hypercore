# viewer v2

Three zones. Left: the intent panel — pending endorsements first (amber, with
an endorse button that asks once before spending judgment), then open work,
all statements grouped by segment in canonical order, recent folds. Center:
the graph — each work is a box containing its operations (membership is
nesting; contains edges are not drawn), only combinators draw as labeled
edges, layout is deterministic so the mental map survives a reload, zoom is
buttons plus a gentle wheel. Right: details — statements show owner, links,
and the reverse index (bound by / produced by); operations show roles,
verdicts with grounds, and material, with inline bodies fetched on demand.

The server gained GET /api/material/ID and POST /api/endorse. Endorse is the
operator acting (endorsement.md): the machine never calls it.
