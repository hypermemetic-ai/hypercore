# decomposition notes

These notes preserve the durable design ideas from the older decomposition findings.
They are not current methodology requirements.

## parent as container

A decomposing parent should remain the conceptual container for the children it creates.
The parent holds the whole purpose. The children hold bounded parts. The important
question is whether the assembled result still means what the parent needed it to mean.

The useful phrase from the older notes is: children come home.

## assembled whole

Local coherence is not enough. A set of children can each make sense alone while failing
to compose into the intended whole.

The parent is where integration pressure returns. It is the place to ask whether the
parts still add up to the purpose.

## strict carve

The older notes favored a strict carve: if a parent decomposes a whole into parts, the
parts are not casual suggestions. They are the shape of the reasoning. Changing that
shape means the parent understanding changed too.

Keep the idea, not the old machinery around enforcing it.

## no invented children

An unmaterialized child is only a possibility. Do not fill it with imagined content just
because the parent has a slot for it.

## deferred specifics

The older notes contained many implementation proposals. Those are deliberately gone. The
remaining idea is conceptual: decomposition must preserve the whole, not merely split the
work.
