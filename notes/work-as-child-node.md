# work-as-child-node notes

These notes preserve the durable idea from the older work-node collapse findings. They
are not current methodology requirements.

## two kinds, not three

The real structural distinction is between leaf material and child nodes.

A temporary effort does not need to be a third kind. If it needs its own boundary, it can
be a child node. If it does not, it can remain ordinary material or conversation.

## empty material

A node with no leaf material is still a node. Seeing only `intent/` does not create a new
category.

## frame contents

The older corpus treated some temporary work descriptions as if they were special
methodology folders. The better idea is simpler: a node chooses its own segments, and
temporary reasoning can be represented as intent, material, or notes according to what it
actually is.

Do not pre-decide that every temporary artifact is intent. Do not pre-decide that every
temporary artifact is material. Keep the split honest.

## appearance

The old notes explored visible markers for temporary nodes. That was an implementation
detail. The retained idea is only that temporary material should be legible enough that a
reader does not confuse it with a settled child.
