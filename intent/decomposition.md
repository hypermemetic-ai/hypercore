# decomposition

Large meaning can be decomposed across child nodes.

A child node is a real node, not a special work type.

A parent can hold a whole purpose while children hold bounded parts.

The value of decomposition is not that parts can be handled separately. The value is that
separation makes the whole easier to reason about without losing the whole.

Children remain bounded by the parent statements that explicitly reach them.

A child should not be judged coherent only in isolation when the parent purpose depends on
the assembled result.

When children return meaning to the parent, the parent is responsible for whether the
assembled whole still satisfies the purpose that justified the decomposition.

Bad decomposition hides integration risk. Good decomposition makes integration risk
visible.

The notes in `notes/` preserve older decomposition thinking as design material, not as
current doctrine.
