# statements

a segment's intent document is that segment's intended reality, written as plain statements.
a statement is declarative and strong enough to be wrong; it is never hedged to be safe.
behavior and dependency are written the same way; the prose distinguishes neither.
one name, one concept.
every statement has an owner: the operator or the machine.
ownership is the right to change a statement, not a license to break it.
operator and machine are both bound by coherence; work that fails the sweep is turned away whoever makes it.
the operator's intent is usually open; the machine settles what is left open.
a freedom is taken, not declared: the operator leaves a choice unmade, the machine settles it, and the machine statement is the record.
operator statements form the body of a segment's intent document; machine statements sit below a `## machine` heading.
every statement is falsifiable and carries a check on the code that holds it true.
a statement that is neither ownable nor checkable is turned away.

## machine
the machine section of a segment is introduced by the heading `## machine`.
